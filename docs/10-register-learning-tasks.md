# 注册功能学习任务清单

## 目标
- 用最短路径独立完成一个可用的注册接口
- 理解完整链路：参数校验 -> 业务校验 -> 密码加密 -> 数据入库 -> 统一异常返回
- 通过最小测试保证功能稳定

## 使用方式
- 按顺序执行任务并勾选
- 每完成一项，马上做对应验收
- 遇到问题先记录，再回到当前步修复，不要跳步

## 阶段 A：先跑通现有注册链路（30 分钟）
- [ ] A1. 启动后端服务并确认接口可访问
- [ ] A2. 用 Postman 或 curl 调用注册接口
- [ ] A3. 记录成功响应结构（code、message、data）
- [ ] A4. 人为传入非法参数，观察错误响应

验收标准
- 能得到一次成功注册响应
- 能得到一次参数校验失败响应

## 阶段 B：自己手写最小注册实现（90 分钟）
- [ ] B1. 新建或重写注册请求 DTO 字段：username、password、nickname、email
- [ ] B2. 给 DTO 增加参数校验注解
- [ ] B3. 在 Controller 中接入注册接口并启用参数校验
- [ ] B4. 在 Service 中实现 createUser 基础流程
- [ ] B5. 返回统一响应体

验收标准
- 接口能正常创建用户
- 参数不合法会被拦截，不进入核心业务

## 阶段 C：补齐关键业务规则（60 分钟）
- [ ] C1. 用户名重复检查
- [ ] C2. 密码使用 PasswordEncoder 加密
- [ ] C3. 设置默认值：role=USER、status=1
- [ ] C4. 注册成功后返回 accessToken 和 refreshToken

验收标准
- 数据库中密码不是明文
- 重复用户名返回业务错误，而不是 500

## 阶段 D：异常和数据库兜底（45 分钟）
- [ ] D1. 给注册失败场景定义业务错误码
- [ ] D2. 使用全局异常处理器统一返回错误结构
- [ ] D3. 确认 user 表唯一索引有效（username 必须唯一）

验收标准
- 错误响应结构稳定，前端可直接消费
- 并发或重复提交不会插入重复用户名

## 阶段 E：最小测试闭环（60 分钟）
- [ ] E1. 成功注册测试
- [ ] E2. 用户名重复测试
- [ ] E3. 参数非法测试

验收标准
- 3 条测试可重复执行并通过
- 覆盖 1 条成功路径 + 2 条失败路径

## 提交前自检
- [ ] 注册接口路径、请求体、返回体与文档一致
- [ ] 任何场景都不会返回明文密码
- [ ] 异常都能被统一处理
- [ ] 关键流程有日志但不包含敏感信息
