# Learn 学习执行层（轻量版）

## 目标
把学习流程从“文档很全但执行很重”，改成“每天能推进、每次都可验证”。

## 你只需要记住 3 件事
1. 先看 `INDEX.md`：确定当前阶段和下一步。
2. 只做一个 Step 的 `00/01/02`：理解、执行、验收。
3. 每次结束只补最小证据：命令、输出、结论。

## 新学习节奏（推荐）
- 每个 Step 控制在 2-4 小时。
- 每次只追求一个“最小闭环”（一个接口或一条链路）。
- 一天结束前必须完成一次 5 分钟验收。

## 目录分工
- `INDEX.md`：主路线图（含先后顺序和里程碑）。
- `INFRA-BOOT.md`：基础设施前置轨（Redis / MinIO / Meilisearch）。
- `Sxx/*/00-overview.md`：本步到底要学会什么。
- `Sxx/*/01-task-board.md`：今天要做的最小任务。
- `Sxx/*/02-acceptance.md`：5 分钟验收脚本。

## 和 docs 的关系
- `docs/`：稳定规范与契约。
- `learn/`：执行路线与学习证据。

## 完成标准（简化）
- 至少 1 条成功路径通过。
- 至少 1 条失败路径通过。
- 至少 1 条构建或测试命令通过。
