# Learn Index（Step 驱动）

## 总览
- 组织方式：`S01` 到 `S08`，按能力构建顺序推进。
- 通过标准：每个 Step 的命令验收全部通过，且复盘记录完整。

| Step | 主题 | 当前状态 | 目标能力 | 主要产出 |
|---|---|---|---|---|
| S01 | 开发基线与分层认知 | 未开始 | 架构边界与环境能力 | 分层认知图、环境验收记录 |
| S02 | 认证与安全链路 | 未开始 | Spring Security + JWT 基础 | 登录/注册链路验收、错误码清单 |
| S03 | 文章读取链路 | 未开始 | 查询建模与分页/详情设计 | 文章列表+详情接口验收 |
| S04 | 文章写入与管理链路 | 未开始 | 写路径、权限、一致性 | 文章 CRUD 验收与回归脚本 |
| S05 | 评论域与树形结构 | 未开始 | 树形数据与规则编排 | 评论策略与树形查询验收 |
| S06 | 搜索索引与查询链路 | 未开始 | 索引一致性与检索接口 | 搜索链路压测与高亮验证 |

## 使用清单
- [ ] 按顺序完成 Step，不跳过验收。
- [ ] 每个 Step 至少补 1 条架构 ADR 记录。
- [ ] 每个 Step 完成后更新状态为“进行中/已完成”。
- [ ] 每两步做一次跨 Step 复盘，识别重复问题。
