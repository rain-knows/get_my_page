- 主策略：先补基础设施最小能力，再推进业务链路。
- 通过标准：每个 Step 完成“成功 + 失败 + 构建/测试”三项最小验收。

## 建议开发顺序（重排后）

### P0：INFRA-BOOT（必须先做）
- Redis：用于 token 黑名单与缓存基线。
- MinIO：用于上传能力基线。
- Meilisearch：用于搜索索引与查询基线。
- 通过标准：三组件本地可访问，基础读写操作可跑通。

### M0：先收口认证链路（S02）
- 目标：把认证从“可用”提升到“可闭环验收”。
- 必做：补全 `POST /api/auth/logout` 的真实登出策略（黑名单或等效机制）。
- 必做：在认证过滤链中加入黑名单校验，确保登出后旧 token 不再可用。
- 必做：补齐成功/失败路径验收与证据归档（登录成功、错误密码、refresh 成功/失败、登出后访问失败）。

### M1：文章读取链路（S03）
- 目标：先打通前台可见价值。
- 后端优先：实现 `GET /api/posts` 与 `GET /api/posts/{slug}`，完成分页与不存在资源错误路径。
- 前端联动：把 `blog` 列表页与详情页从 mock 数据切到真实接口。

### M2：文章写入与管理链路（S04）
- 目标：形成“可发布内容”的最小闭环。
- 后端优先：实现 `POST/PUT/DELETE /api/posts` 与管理员权限校验。
- 配套：补齐分类/标签合法性校验与写后缓存失效策略。
- 前端联动：管理端文章页面接入创建、编辑、删除。

### M3：评论链路（S05）
- 目标：打通内容互动能力。
- 后端优先：实现 `POST /api/comments` 与 `GET /api/posts/{postId}/comments`（树形结构）。
- 前端联动：在文章详情页接入评论发布与树形展示。

### M4：搜索链路（S06）
- 目标：打通检索与索引一致性。
- 后端优先：实现 `GET /api/search`，并补全 `SearchClient` 索引写入。
- 前端联动：搜索页由占位数据切换为真实检索结果与高亮渲染。

### M5：上传链路（S07）
- 目标：为封面/编辑器资源提供稳定上传能力。
- 后端优先：补全 `StorageClient` 的 MinIO 上传、类型与大小校验。
- 前端联动：管理端文章表单接入上传与失败提示。

### M6：稳定性与预发布（S08）
- 目标：在合并主干前建立质量门禁。
- 执行：最小回归矩阵、容器编排校验、依赖故障注入与预发布手册归档。
